# MedQueue — מערכת ניהול תורים לבית רופאים

מערכת מלאה לניהול תורים: **קיוסק**, **מסך תצוגה**, **הקראת שמות**, **הדפסת כרטיסים**, **מעבר בין חדרים** ו**לוח בקרה** לצוות.

## התקנה והרצה

```bash
npm run setup
npm run dev
```

**דרישה:** Node.js 22.5 ומעלה (כולל 24). אין צורך ב-`better-sqlite3` — משתמשים ב-SQLite המובנה של Node.

- **דף בית:** http://localhost:5173
- **קיוסק:** http://localhost:5173/kiosk
- **ניהול:** http://localhost:5173/admin
- **מסך לובי:** http://localhost:5173/display
- **מסך חדר (דוגמה רופא 1):** http://localhost:5173/display/room/3

לפרודקשן מקומי:

```bash
npm run install:all
npm run build
npm start
```

הכל ירוץ על פורט **3001** (שרת + קבצים סטטיים). לפיתוח עם שחרור פורטים: `npm run start:local`.

### פריסה ב-Render (ענן)

ראה **[RENDER-DEPLOY.md](./RENDER-DEPLOY.md)** — התקנה אוטומטית דרך `render.yaml` (בנייה + דיסק ל-SQLite).

ריפו: https://github.com/stech-il/MedQueue

## רכיבי המערכת

| רכיב | תיאור |
|------|--------|
| **קיוסק** | המטופל בוחר שירות, מזין שם (אופציונלי), מקבל מספר תור והדפסה אוטומטית |
| **ניהול** | צוות בוחר חדר, קורא תור, מעביר בין חדרים, מסיים טיפול |
| **מסך לובי** | מציג את המטופל הנוכחי בכל חדר רופא/מעבדה |
| **מסך חדר** | תצוגה גדולה + רשימת המתינים + הקראה קולית בעברית |
| **הדפסה** | הדפסה אוטומטית לכרטיס (80mm). להדפסה ישירה למדפסת ברירת מחדל **בלי** חלון בחירה: הפעל Chrome במצב קיוסק עם `--kiosk-printing` (ראה למטה) |
| **זמן אמת** | עדכון מיידי בכל המסכים דרך WebSocket |

## זרימת עבודה טיפוסית

1. מטופל לוקח תור ב**קיוסק** → מתקבל כרטיס מודפס (למשל `A12`).
2. התור נכנס ל**חדר ההמתנה** (או ישירות לחדר השירות).
3. ב**ניהול**, אחות/רופא בוחרת חדר ולוחצת **קרא הבא**.
4. **מסך החדר** מציג את המספר בגדול ו**מקריא** את השם (אם הוזן).
5. לאחר הטיפול — **העברה** לחדר אחר (מעבדה, רופא נוסף) או **סיום**.

## חדרים ושירותים (ברירת מחדל)

- חדרים: קבלה, המתנה, רופא 1, רופא 2, מעבדה
- שירותים: רופא כללי (A), רופא מומחה (B), מעבדה (L), קבלה (R)

ניתן להרחיב דרך API (`POST /api/rooms`, `POST /api/services`).

## הגדרות

- שם המרפאה: `PUT /api/settings` עם `{ "clinic_name": "שם המרפאה" }`
- מסד נתונים: `server/data/medqueue.db` (SQLite)

## חומרה מומלצת

- **קיוסק:** מסך מגע + מדפסת קבלות USB
- **תצוגה:** מסך טלוויזיה / מוניטור ב-F11 (מסך מלא)
- **ניהול:** מחשב או טאבלט לצוות

### קיוסק — הדפסה אוטומטית (ללא לחיצה על «הדפס»)

1. המטופל מזין טלפון ות.ז., **בוחר קופה** — התור נוצר והכרטיס מודפס מיד.
2. **מומלץ (Windows):** בניהול → הגדרות מרפאה → **הדפסה מהשרת** — ללא חלון דפדפן. הגדר מדפסת קבלות כ**ברירת מחדל** (או ציין שם מדפסת בהגדרות).
3. **חלופה:** הדפסה דרך הדפדפן — הפעל `scripts\start-kiosk.bat` או Chrome עם `--kiosk-printing`:

```text
"C:\Program Files\Google\Chrome\Application\chrome.exe" --kiosk --kiosk-printing http://localhost:5173/kiosk
```

בדפדפן רגיל בלי `--kiosk-printing` עדיין עלול להופיע חלון הדפסה.

## טכנולוגיות

- Node.js + Express + SQLite + Socket.io
- React + Vite
- Web Speech API להקראה בעברית

### הקראה ברורה (חשוב)

1. פתח **מסך תצוגה** (`/display` או `/display/room/X`) — ההקראה מתבצעת **רק שם**, לא בניהול.
2. ההקראה מופעלת אוטומטית בטעינת המסך (במצב קיוסק של Chrome מומלץ).
3. ב-Windows: התקן **שפה עברית + דיבור (TTS)** — הגדרות → שפה ואזור → עברית → אפשרויות → דיבור.
4. השתמש ב-**Chrome** או **Edge** (תמיכה טובה יותר מ-Firefox).
